Последовательный поиск

Последовательный поиск — поиск в конечном итераторе. Сразу же с порога коллизия в терминологии. Здесь важно, что мы можем двигаться только вперед (в некоторых экзотических случаях и назад), но произвольного доступа у нас нет. То есть мы не можем вытащить элемент с конкретным номером.

Пусть у нас есть итератор из ключей K1,K2,…,KnK_1, K_2, \dotsc, K_nK1​,K2​,…,Kn​ (на их структуру нам пока наплевать). Нам нужно понять, есть ли в этом наборе ключ ttt, и, если он есть, найти его номер.

Банальная идея «начать с начала и искать, пока не найдешь» не так уж и плоха. Код, реализующий этот подход

function search(iterable x, target) -> bool: for index, element of enumerate(x): if element == target: return index return not found

Этот же алгоритм применим и для массивов (в общем случае последовательностей), про внутреннюю структуру которых мы ничего не знаем. Более того, при отсутствующей информации о структуре это самый эффективный алгоритм.

Понятно, что такой алгоритм работает за время O(n)O(n)O(n).

Показатель времени работа такого алгоритма — количество проверок (сравнений) CCC. Если позиция целевого элемента равновероятно может равняться любому числу от 111 до nnn, то ожидаемое количество проверок

E⁡C=∑i   ⁣=   ⁣1ni⋅P⁡(i)=1n∑i   ⁣=   ⁣1ni=n+12\expect C = \sum\limits_{i \;\! = \;\! 1}^n i \cdot \prob(i) = \frac{1}{n} \sum\limits_{i \;\! = \;\! 1}^n i = \frac{n+1}{2}EC=i=1∑n​i⋅P(i)=n1​i=1∑n​i=2n+1​

Упорядоченная последовательность

Пусть все элементы нашего итератора упорядочены, то есть xj>xix_j > x_ixj​>xi​ для j>ij > ij>i.

Мы хотим найти элемент ttt и его позицию. Если на какой-то итерации iii элемент xix_ixi​ оказался больше ttt, то дальше продолжать смысла нет: из-за упорядоченности итератора все остальные

function search(iterable x, target): for index, element of enumerate(x): if element == target: return index if element > target: return not found return not found

Временная сложность этого алгоритма совпадает с временной сложностью обычного алгоритма последовательного поиска. Отличие здесь состоит лишь в том, что для отсутствующих в итераторе элементов этот факт отсутствия может быть установлен гораздо раньше.

Частота обращений

Мы предполагали, что все аргументы последовательного поиска равновероятны. Вообще это не так. Распределение вероятностей может быть любым.

Пусть вероятность запроса на поиск ключа KjK_jKj​ равна pjp_jpj​. И пусть ответ на наш запрос всегда есть, то есть искомый элемент точно в итераторе.

∑j   ⁣=   ⁣1npj=1\sum\limits_{j \;\! = \;\! 1}^n p_j = 1j=1∑n​pj​=1

Математическое ожидание количества проверок в алгоритме последовательного поиска равно

E⁡C=∑j   ⁣=   ⁣1nj⋅pj\expect C = \sum\limits_{j \;\! = \;\! 1}^n j \cdot p_jEC=j=1∑n​j⋅pj​

Количество сравнений E⁡C\expect CEC не может быть больше энтропии.

Пусть есть какое-то «идеальное» распределение вероятностей q1,q2,…,qnq_1, q_2, \dotsc, q_nq1​,q2​,…,qn​

Если нам подвластна структура итератора, то минимальное ожидаемое время работы достигается при

p1⩾p2⩾⋯⩾pnp_1 \ge p_2 \ge \dotsb \ge p_np1​⩾p2​⩾⋯⩾pn​

То есть, нам нужно переупорядочить элементы так, чтобы вероятности запроса на них убывали.

Вообще, это все немного противоречит самой концепции итераторов, но допустим, что мы с самого начала определяем структуру итератора, а потом этим пользуемся.

Распределение Зипфа

Более типично для реальных ситуаций распределение Зипфа:

p1=1Hnp2=12Hn⋯pi=1iHn⋯pn=1nHnp_1 = \frac{1}{H_n} \quad p_2 = \frac{1}{2 H_n} \quad \cdots \quad p_i = \frac{1}{i H_n} \quad \cdots \quad p_n = \frac{1}{n H_n}p1​=Hn​1​p2​=2Hn​1​⋯pi​=iHn​1​⋯pn​=nHn​1​

Джордж Кингсли Зипф (George Kingsley Zipf), изучая статистические закономерности в естественных языках, обнаружил, что nnn-е по частоте слово встречается с частотой, примерно обратно пропорциональной nnn.

По этому распределению pi=c/ip_i = c/ipi​=c/i. Осталось найти константу из условия нормировки

∑i   ⁣=   ⁣1npi=1  ⟹  c⋅Hn=1  ⟹  c=1Hn\sum\limits_{i \;\! = \;\! 1}^n p_i = 1 \implies c \cdot H_n = 1 \implies c = \frac{1}{H_n}i=1∑n​pi​=1⟹c⋅Hn​=1⟹c=Hn​1​

При таком распределении ожидаемое количество сравнений

E⁡C=nHn∼nln⁡n\expect C = \frac{n}{H_n} \sim \frac{n}{\ln n}EC=Hn​n​∼lnnn​

Получается, что это почти в 12ln⁡n\frac{1}{2} \ln n21​lnn раз быстрее, чем при предположении о равномерном распределении.

Распределение Парето

Еще более близкое к реальному распределение Парето (наиболее известное как правило 80-20), которое на самом деле возникло в процессе изучения спроса и транзакций. Человеческим языком его можно сформулировать так «80% транзакций работают с 20% данных».

Коротко, можно записать это распределение (с упорядоченными вероятностями) формулой

p1+p2+⋯+p0.2⋅mp1+p2+⋯+pm=0.8\frac{p_1 + p_2 + \dotsb + p_{0.2 \cdot m}}{p_1 + p_2 + \dotsb + p_m} = 0.8p1​+p2​+⋯+pm​p1​+p2​+⋯+p0.2⋅m​​=0.8

На практике, значения 20% и 80% могут чуть-чуть отличатся, поэтому целесообразно ввести коэффициент

θ=log⁡0.8log⁡0.2\theta = \frac{\log 0.8}{\log 0.2}θ=log0.2log0.8​

Из формулы-определения следуют формулы для вероятностей в терминах коэффициента θ\thetaθ

p1=1nθp2=2θ−1nθ⋯pi=iθ−(i−1)θnθ⋯pn=nθ−(n−1)θnθp_1 = \frac{1}{n^\theta} \quad p_2 = \frac{2^\theta - 1}{n^\theta} \quad \cdots \quad p_i = \frac{i^\theta - (i-1)^\theta}{n^\theta} \quad \cdots \quad p_n = \frac{n^\theta - (n-1)^\theta}{n^\theta}p1​=nθ1​p2​=nθ2θ−1​⋯pi​=nθiθ−(i−1)θ​⋯pn​=nθnθ−(n−1)θ​

Теперь можно найти ожидаемое количество сравнений

E⁡C=∑i   ⁣=   ⁣1ni⋅iθ−(i−1)θnθ=1nθ∑i   ⁣=   ⁣1ni⋅(iθ−(i−1)θ)=n+1−1nθ∑i   ⁣=   ⁣1niθ\expect C = \sum\limits_{i \;\! = \;\! 1}^n i \cdot \frac{i^\theta - (i-1)^\theta}{n^\theta} = \frac{1}{n^\theta} \sum\limits_{i \;\! = \;\! 1}^n i \cdot \big( i^\theta - (i-1)^\theta \big) = n + 1 - \frac{1}{n^\theta} \sum\limits_{i \;\! = \;\! 1}^n i^\thetaEC=i=1∑n​i⋅nθiθ−(i−1)θ​=nθ1​i=1∑n​i⋅(iθ−(i−1)θ)=n+1−nθ1​i=1∑n​iθ

Используя асимптотическую оценку суммы ∑i   ⁣=   ⁣1niθ∼nθ+1θ+1+nθ2+O(nθ−1)\sum\limits_{i \;\! = \;\! 1}^n i^\theta \sim \frac{n^{\theta+1}}{\theta+1} + \frac{n^\theta}{2} + O(n^{\theta-1})i=1∑n​iθ∼θ+1nθ+1​+2nθ​+O(nθ−1) получаем, что

E⁡C=n+1−1nθ∑i   ⁣=   ⁣1niθ=θθ+1⋅n+12+O(1n)\expect C = n + 1 - \frac{1}{n^\theta} \sum\limits_{i \;\! = \;\! 1}^n i^\theta = \frac{\theta}{\theta+1} \cdot n + \frac{1}{2} + O \left( \frac{1}{n} \right)EC=n+1−nθ1​i=1∑n​iθ=θ+1θ​⋅n+21​+O(n1​)

Результат довольно хороший. При коэффициенте θ=log⁡0.20.8≈0.139\theta = \log_{0.2}{0.8} \approx 0.139θ=log0.2​0.8≈0.139 количество сравнений оценивается по только что выведенной формуле как 0.5+0.122037⋅n0.5 + 0.122037 \cdot n0.5+0.122037⋅n.
