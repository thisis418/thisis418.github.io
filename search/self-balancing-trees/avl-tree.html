Фактор балансировки

Все проблемы несбалансированных деревьев из-за высоты. Точнее, из-за большой разности высот поддеревьев. Давайте на основе этой разности введем фактор балансировки.

Для каждой вершины vvv определим height balance factor — разницу высот правого и левого поддеревьев.

bf⁡v=height⁡(v.left)−height⁡(v.right)\bf v = \height(v.\code{left}) - \height(v.\code{right})bfv=height(v.left)−height(v.right)

Если balance factor у каких-то вершин по модулю большой, значит наше дерево перекашивает, и высота становится неприемлемой.

Балансировка

Проще всего реализовать балансировку при требовании ∣bf⁡v∣⩽1|\bf v| \le 1∣bfv∣⩽1 для всех вершин vvv.

Итак, после операции вставки или удаления вершины дерево может перестать быть сбалансированным, то есть у какой-то вершины ∣bf⁡v∣>1|\bf v| > 1∣bfv∣>1. Надо научиться возвращать дереву баланс.

Важное наблюдение заключается в том, что после вставки или удалении вершины balance factor может измениться только тех у вершин, которые лежат на пути от корня до вставленной или удаленной вершины. И изменяется он на 111 или на −1-1−1. Это позволяет вынести балансировку дерева в две процедуры, которые будут вызываться после каждой вставки и после каждого удаления.

Поворот

Обычный поворот относительно вершины выравнивает высоты её поддеревьев. После поворота высота дерева не увеличивается, то есть уменьшается на 111 или остается неизменной.

Если bf⁡A=2\bf A = 2bfA=2, то осуществляем правый поворот. Если bf⁡A=−2\bf A = -2bfA=−2, то левый поворот.

function rotate_right(node a): node b = a.left a.left = b.right b.right = a height adjustment a height adjustment b
function rotate_left(node a): node b = a.right a.right = b.left b.left = a height adjustment a height adjustment b

Большой поворот

В результате обычного поворота balance factor у вершины B может стать 222 или −2-2−2, и дерево снова будет разбалансированным. Эту проблему решают большие повороты влево и вправо, которые состоят из двух обычных поворотов: у вершины и у ее ребёнка.

function big_rotate_left(node a): rotate_right(a.right) rotate_left(a)
function big_rotate_right(node a): rotate_left(a.left) rotate_right(a)
