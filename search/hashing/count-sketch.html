Count sketch

Пусть у нас есть поток данных (stream) s1,s2,s3,…,si,…s_1, s_2, s_3, \dotsc, s_i, \dotscs1​,s2​,s3​,…,si​,…, при этом все данные — элементы какого-то универсума U\UUUU. Мы хотим уметь считать, сколько раз элемент xxx встретился в этом потоке к моменту времени ttt, то есть уметь считать значение функции

ft(x)=∑i=1tI⁡(si=x)f_t(x) = \sum\limits_{i=1}^t \indic(s_i = x)ft​(x)=i=1∑t​I(si​=x)

Будем решать задачу для фиксированной временной границы ttt.

Задача подсчета

Дан поток данных S=s1,s2,s3,…,sn−1,snS = s_1, s_2, s_3, \dotsc, s_{n-1}, s_nS=s1​,s2​,s3​,…,sn−1​,sn​. Можно считать, что мы сделали временной срез при t=nt = nt=n.

Нам нужно оценивать количество вхождений элемента xxx в этот поток SSS — значение функции f(x)f(x)f(x).

f(x)=∑i=1nI⁡(si=x)f(x) = \sum\limits_{i=1}^n \indic(s_i = x)f(x)=i=1∑n​I(si​=x)

Но ответы на запросы лояльные. Мы даем ответ с точностью ε\epsilonε, при этом позволяем себе ошибаться с вероятностью δ\deltaδ. Формально, P⁡(f^(x)⩽f(x)+εn)⩾1−δ\prob \big( \hat f (x) \le f(x) + \epsilon n \big) \ge 1 - \deltaP(f^​(x)⩽f(x)+εn)⩾1−δ, где f^(x)\hat f (x)f^​(x) — предсказанное нами количество вхождений.

А ещё нам нужно уметь обновлять поток — добавлять и удалять элементы.

Count-Min sketch

Для решения задачи подсчета можно использовать структуру данных Count-Min sketch. Она применима только тогда, когда в поток элементы только добавляются.

Будем использовать хеширование.

Пусть d=⌈ln⁡1/δ⌉d = \lceil \ln 1/\delta \rceild=⌈ln1/δ⌉ — глубина aka количество хеш-функций, и w=⌈e/ε⌉w = \lceil e/\epsilon \rceilw=⌈e/ε⌉ — размер хеш-пространства. Счётчики будем хранить в двумерном массиве c∈N0d×wc \in \NN_0^{d \times w}c∈N0d×w​ высоты ddd и ширины www. Изначально массив ccc проинициализирован нулями.

Создадим еще ddd независимых хеш-функций h1,h2,…,hd ⁣:U↠{1,2,⋯ ,w}h_1, h_2, \dotsc, h_d \colon \UUU \surjto \{1, 2, \dotsb, w\}h1​,h2​,…,hd​:U↠{1,2,⋯,w}.

const real delta = 0.01 const real epsilon = 0.01 const int w = ceil(e / epsilon) const int d = ceil(ln(1 / delta)) const array h[d] array[int] c[d][w]

Обновление

При получении запроса на добавления в «счётчик» элемента xxx мы для каждой строки iii массива ccc увеличиваем значение соответствующего счётчика hi(x)h_i(x)hi​(x) на 111.

function add(self, element): for i = 0; i < d; i++: column = h[i](element) c[i][column] += 1

Оценка

Для получения оценки количества вхождений элемента xxx в поток данных нам надо посчитать минимум по всем счётчикам, включающим в себя этот элемент:

f^(x)=min⁡1   ⁣⩽   ⁣i⩽dc[i][hi(i)]\hat f (x) = \min\limits_{1 \;\! \le \;\! i \le d} c[i][h_i(i)]f^​(x)=1⩽i⩽dmin​c[i][hi​(i)]

Все показатели счетчиков получаются завышенными, ведь у любой хеш-функции неизбежно будут коллизии. Когда мы берем минимум, мы выбираем наименее завышенную оценку.

Доказательство того, что структура работает, и работает хорошо.

Начнем с доказательства завышенности оценок. Для любой строки iii и любого элемента xxx

c[i][hi(x)]=f(x)+∑y≠xhi(y)=hi(x)f(y)⩾f(x)c[i][h_i(x)] = f(x) + \sum\limits_{\substack{y \;\! \neq \;\! x \\ h_i(y) = h_i(x)}} f(y) \ge f(x)c[i][hi​(x)]=f(x)+y=xhi​(y)=hi​(x)​∑​f(y)⩾f(x)

Хеш-функции hih_ihi​ «хорошие», то есть они свои входы равномерно распределяют по выходам.

Но это только для одной строки. А при получении оценки мы выбираем минимум, значит, общая ошибка превысит εn\epsilon nεn, только тогда, когда переоценку дадут все ddd строк.
