Хеширование

В главах «Простой поиск», «Самобалансирующиеся деревья», «Кусты» и «Боры» мы разбирали исключительно методы поиска, основанные на сравнении ключей целиком или по частям. Любой подход, основанный на сравнении, требует много операций как минимум самого сравнения.

Существует принципиально иной путь — хеширование. Основная идея заключается в том, чтобы отказаться от последовательного просмотра данных, а на основе искомого ключа KKK сразу вычислять значение функции h(K)h(K)h(K), которая укажет место хранение записи, информацию по ключу или любую другую информацию, которую мы ищем.

Хеш-функции

Хеш-функция

Хеш-функция — сюръективная функция, отображающее какой-то объект в фиксированное множество из mmm элементов, часто представляющее собой просто {0,1,2,…,m−2,m−1}\{0, 1, 2, \dotsc, m-2, m-1\}{0,1,2,…,m−2,m−1}

hash⁡ ⁣:U↠{0,1,2,…,m−2,m−1}\hash \colon \UUU \surjto \{0, 1, 2, \dotsc, m-2, m-1\}hash:U↠{0,1,2,…,m−2,m−1}

Преобразование, выполняемое хеш-функцией называется хешированием.

Рассмотрим нереалистичный, но показательный пример. Пусть у нас mmm ключей K1,K2,…,KmK_1, K_2, \dotsc, K_mK1​,K2​,…,Km​. Вместо организации их в список или дерево, мы разместим их в обычном массиве a.

Сами ключи могут быть очень большими числами, строками, структурами и так далее, и не могут в чистом виде выступать в качестве индекса массива. Поэтому нам придётся придумывать хеш-функцию hhh, которая будет по заданному ключу KKK вычислять его индекс в массиве a, то есть a[h(K)]=K\code{a} \bigl[ h(K) \bigr] = Ka[h(K)]=K.

Получается, что мы теперь можем искать за чистое время O(1)O(1)O(1) — вычислить h(K)h(K)h(K) и перейти по адресу. Добавив особые значения none и чуть-чуть модифицировав структуру, можем получить не только константный поиск, но и константную вставку и удаления ключей.

Давайте попытаемся отобразить nnn-элементное множество ключей в mmm-элементное множество выхода хеш-функции. Считаем, что m>nm > nm>n. Всего таких отображений существует mnm^nmn, но при этом только m!/(m−n)!m!/(m-n)!m!/(m−n)! из них дают различные значения для различных аргументов. Это ничножно маленькая часть.

Вообще, придумать такую функцию, которая попала бы в эту маленькую в часть приемлемых функций очень-очень сложно. Более того, такая функция должна просто и быстро вычисляться.

Универсум U\UUUU очень большой, даже бесконечный, а выходное множество очень маленькое. Значит, по принципу Дирихле, существуют два таких объекта aaa и bbb, что a≠ba \neq ba=b и при этом hash⁡a=hash⁡b\hash a = \hash bhasha=hashb.

Коллизии

Коллизия — двойка неравных входных данных для хеш-функций, у которых хеши одинаковые.

(a,b) — коллизия   ⁣≡≡def   ⁣a≠b∧hash⁡(a)=hash⁡(b)(a, b) ~\text{--- коллизия} \defequiv a \neq b \land \hash(a) = \hash(b)(a,b) — коллизия≡≡defa=b∧hash(a)=hash(b)

Коллизии — штуки плохие. Нам бы очень хотелось придумать такую хеш-функцию, которая вообще не имела бы коллизий.

Пусть мы всё так же отображаем nnn-элементное множество ключей в mmm-элементное множество хешей. Сколько может быть коллизий? Предположим, что хеш-функция равномерная, то есть все mmm выходов равновероятны.

Введём индикаторы для коллизий между элементами

Полиномиальное хеширование

Мы научились хешировать числа. Теперь мы хотим научиться хешировать последовательности чисел. Точнее, формально, мы хотим научиться хешировать строки из языка A∗A^*A∗, где каждый символ в алфавите AAA закодирован каким-то числом.

Строку sss будем представлять как упорядоченный набор символов sis_isi​, которые будем воспринимать как числа.

s=s0 s1 s2 s3⋯sn−3 sn−2 sn−1 sns = s_0 \, s_1 \, s_2 \, s_3 \dotsm s_{n-3} \, s_{n-2} \, s_{n-1} \, s_ns=s0​s1​s2​s3​⋯sn−3​sn−2​sn−1​sn​

Полиномиальный хеш

Прямым полиномиальным хешем строки sss называется число

hash⁡(s)=∑i=0nsi⋅xi mod m=(s0+s1⋅x+s2⋅x2+⋯+sn−1⋅xn−1+sn⋅xn) mod m\hash(s) = \sum\limits_{i=0}^{n} s_i \cdot x^i \bmod m = \big( s_0 + s_1 \cdot x + s_2 \cdot x^2 + \dotsb + s_{n-1} \cdot x^{n-1} + s_n \cdot x^n \big) \bmod mhash(s)=i=0∑n​si​⋅ximodm=(s0​+s1​⋅x+s2​⋅x2+⋯+sn−1​⋅xn−1+sn​⋅xn)modm

Обратным полиномиальным хешем строки sss называется число

hash⁡(s)=∑i=0nsi⋅xn−i mod m=(s0⋅xn+s1⋅xn−1+s2⋅xn−2+⋯+sn−1⋅x+sn) mod m\hash(s) = \sum\limits_{i=0}^{n} s_i \cdot x^{n-i} \bmod m = \big( s_0 \cdot x^n + s_1 \cdot x^{n-1} + s_2 \cdot x^{n-2} + \dotsb + s_{n-1} \cdot x + s_n \big) \bmod mhash(s)=i=0∑n​si​⋅xn−imodm=(s0​⋅xn+s1​⋅xn−1+s2​⋅xn−2+⋯+sn−1​⋅x+sn​)modm

Здесь xxx — произвольное число, большее размера алфавита, а mmm — достаточно большой модуль, взаимнопростый с xxx.

Считаются прямые и обратные полиномиальные хеши почти в лоб. Сами значения многочленов можно быстро считать с помощью индуктивных вычислений, ведь сам многочлен — индуктивная функция.

Обратный хеш еще и красиво пишется. Однако, он немного непривычен из-за обратного порядка символов.

const int x = ... const int m = ... function forward_hash(string s) -> int: int hash = 0 int monome = 1 for c of s: hash = (hash + monome * c) % m monome = (monome * x) % m return hash function backward_hash(string s) -> int: int hash = 0 for c of s: hash = (hash * x + c) % m return hash

Свойства и быстрый пересчет

Если нам известны хеши двух строк aaa и bbb, то мы можем быстро посчитать хеш конкатенации этих строк ababab.

hash⁡(ab)=hash⁡(a)+x∣a∣⋅hash⁡(b)\hash(ab) = \hash(a) + x^{|a|} \cdot \hash(b)hash(ab)=hash(a)+x∣a∣⋅hash(b)

Отсюда легко выводятся алгоритмы удаления префикса и суффикса строки

Для удаления суффикса достаточно просто вычесть хеш суффикса

hash⁡(a)=hash⁡(ab)−x∣a∣⋅hash⁡(b)\hash(a) = \hash(ab) - x^{|a|} \cdot \hash(b)hash(a)=hash(ab)−x∣a∣⋅hash(b)

А для удаления префикса придется делить по модулю mmm, то есть умножать на обратный по модулю mmm элемент

hash⁡(b)=(hash⁡(ab)−hash⁡(a))⋅(x∣a∣)−1\hash(b) = \big( \hash(ab) - \hash(a) \big) \cdot \big( x^{|a|} \big)^{-1}hash(b)=(hash(ab)−hash(a))⋅(x∣a∣)−1

Использовав две предыдущие формулы, можно получить формулу вычисления любого среза строки.

Нам нужна некоторая подготовка. Если мы хотим для данной строки быстро находить хеш любого её среза, то нам придется предварительно посчитать хеши всех префиксов этой строки. Хранить всё будем в массиве hhh, где hih_ihi​ — хеш префикса, состоящего из iii символов.

const int x = ... const int m = ... string s array[int] h h[0] = 0 # Хеш пустой строки равен 0 for i = 1; i <= length(s); i++: h[i+1] = (h[i] + x ** i * s[i]) % m

Теперь хеш среза sl:rs_{l:r}sl:r​ строки sss легко считается по формуле

hash⁡(sl:r)=(hr−hl)⋅(xl)−1\hash(s_{l:r}) = (h_r - h_l) \cdot \big( x^l \big)^{-1}hash(sl:r​)=(hr​−hl​)⋅(xl)−1
