Двоичная и d-арная куча

Про приоритетные очереди

С помощью линейных структур нельзя реализовать эффективную приоритетную очередь.

Двоичная куча

Давайте создадим бинарное дерево, в котором каждая вершина будет больше своих потомков. При этом, для достижения минимальной высоты ⌊log⁡2n⌋+1\lfloor \log_2 n \rfloor + 1⌊log2​n⌋+1, сделаем дерево полным.

Полное бинарное дерево можно хорошо хранить в массиве h длины nnn (индексация, напомню, с 000). Тогда у элемента с индексом iii детьми являются элементы с индексами 2i+12i+12i+1 и 2i+22i+22i+2, а родителем — элемент с индексом ⌊(i−1)/2⌋\lfloor (i-1)/2 \rfloor⌊(i−1)/2⌋.

По нашему построению видно, что максимальный элемент в куче имеет индекс 000, а минимальный элемент находится в одном из листьев. А так же для всех элементов выполнено «свойство кучи»:

h[i]⩾h[⌊(i−1)/2⌋]для всех i\code{h}[i] \ge \code{h} \bigl[ \lfloor (i-1)/2 \rfloor \bigr] \quad \text{для всех}~ ih[i]⩾h[⌊(i−1)/2⌋]для всех i

Теперь мы можем производить какие-то операции с кучей. А именно, вставлять элементы, изменять элементы и извлекать минимум (про тонкости реализации позже).

После изменения элементов свойство кучи может нарушиться. Поэтому нам надо научиться быстро восстанавливать его.

Восстановление свойства кучи

Пусть какой-то один элемент мешает свойству кучи быть выполненным. Как мы увидим позже, любые операции над кучей в итоге оставляют ровно один плохой элемент.

Если этот элемент слишком большой, то его надо пропихнуть вверх по куче. Если же он наоборот, слишком маленький, то его надо пропихнуть вниз. Вот эти две операции пропихивания вверх и вниз называются по-научному просеиванием.

Просеивание вниз (shift down).

  1. Если просеиваемый элемент меньше, чем его дети, то меняем его с наибольшим из его детей.
  2. Продолжаем его просеивать, пока он не встанет на нормальное место или пока он не станет листом.
function shift_down(mutable array h, int i): const int n = length(h) while 2 * i + 1 < n: left = 2 * i + 1 right = 2 * i + 2 if right < n and h[right] > h[left]: j = right else: j = left break if h[i] >= h[j] swap h[i], h[j] i = j

Если просеиваемый вниз элемент находился на высоте hhh, то выполнение операции потребует не более h−1h-1h−1 обменов и не более 2h−22h-22h−2 сравнений. Константа 222 вылезает потому, что сравниваем мы каждым ребенком.

Просеивание вверх (shift up).

  1. Если просеиваемый элемент меньше, чем его родитель, то меняем его с его родителем.
  2. Продолжаем его просеивать, пока он не встанет на нормальное место или пока он не станет корнем.

Здесь нет никаких ветвлений, в отличие от просеивания вниз, поэтому код будет более красивым

function shift_up(mutable array h, int i): while h[i] > h[(i - 1) / 2]: swap h[i], h[(i - 1) / 2] i = (i - 1) / 2

Обе операции работают за время O(высота кучи)=O(log⁡n)O(\text{высота кучи}) = O(\log n)O(высота кучи)=O(logn).

Вставка

Вставка нового элемента в кучу происходит очень просто: мы добавляем в конец массива h новый элемент и просеиваем его вверх. После просеивания новый элемент займет корректное положение.

Извлечение максимума

При извлечении минимума можно воспользоваться примерно тем же трюком, что и со вставкой. Поместим на место корня элемент −∞-\oo−∞, который гарантированно меньше любого другого элемента. Теперь осталось просто просеять вниз этот элемент.

Можно обойтись и без введения особого элемента, если вспомнить, что листы — наименьшие элементы в куче. Можно на место корня ставить не −∞-\oo−∞, а какой-то лист, например последний. Эффект от просеивания этого элемента вниз будет точно таким же, как и от просеивания −∞-\oo−∞: свойство кучи сохранено, максимум извлечён.

Построение кучи

Представим, что у нас есть неупорядоченный массив, который мы хотим превратить в кучу. Можно действовать в лоб: создать пустую кучу и добавлять туда элементы из массива. Такой подход имеет временную сложность O(nlog⁡n)O(n \log n)O(nlogn).

Но в куче очень много беспорядков. Можно этим воспользоваться. Построить кучу из массива может операция heapify.

Представим, что в исходном массиве уже записано полное бинарное дерево. Оно не удовлетворяет свойству кучи. Давайте просеем вниз все узлы, которые имеют хотя бы одного ребёнка. При этом будем просеивать снизу вверх, то есть начнем с элемента на позиции ⌊n/2⌋\lfloor n/2 \rfloor⌊n/2⌋, а закончим корнем, который находится на позиции 111.

В итоге мы из массива получим нормальную кучу. До просеивания какого-то узла оба его поддерева удовлетворяли свойству кучи. После просеивания этого узла он вместе со своими поддеревьями будет образовывать кучу. Значит, после просеивания всех узлов у нас получится куча.

function heapify(mutable array h): const int n = length(h) for int i = (n-1)/2; i >= 0; i--: sift_down(h, i)

Время работы этой операции O(n)O(n)O(n).

Число вершин на высоте hhh не более ⌈n/2h⌉\lceil n / 2^h \rceil⌈n/2h⌉. Для каждой из них будет вызвана операция просеивания вниз, которая потребует не более hhh обменов и не более 2h2h2h сравнений. Значит, количество сравнений в операции heapify равно

∑h=1⌊log⁡2n⌋+1⌈n2h⌉⋅(2h−2)=2n∑h=1⌊log⁡2n⌋+1h−12h=4n+O(log⁡n)\sum\limits_{h=1}^{\lfloor \log_2 n \rfloor + 1} \Bigl\lceil \frac{n}{2^h} \Bigr\rceil \cdot (2h-2) = 2n \sum\limits_{h=1}^{\lfloor \log_2 n \rfloor + 1} \frac{h-1}{2^h} = 4n + O(\log n)h=1∑⌊log2​n⌋+1​⌈2hn​⌉⋅(2h−2)=2nh=1∑⌊log2​n⌋+1​2hh−1​=4n+O(logn)

А количество обменов в 222 раза меньше, 2n+O(log⁡n)2n + O(\log n)2n+O(logn).

Анализ алгоритмов на куче

Одна из самых важных характеристик кучи — размеры её поддеревьев. Просто так получить все значения, к сожалению нельзя. Проблемы возникают только из-за того, что куча — полное дерево, но не всегда абсолютно сбалансированное.

В куче есть «особый путь», соединяющий корень кучи с последним листом. На картинке я его отметил красным.

Давайте все узлы, лежащие на особом пути, называть особыми. И поддеревья с корнями в особых узлах тоже будем называть особыми.

Все неособые поддеревья являются абсолютно сбалансированными, значит их размер всегда равен 2k−12^k - 12k−1. А количество неособых поддеревьев очень легко вычисляется:

неособых деревьев размера 2k−1 ровно⌊n−2k−12k⌋ штук\text{неособых деревьев размера}~ 2^k-1 ~\text{ровно} \left\lfloor \frac{n - 2^{k-1}}{2^k} \right\rfloor ~\text{штук}неособых деревьев размера 2k−1 ровно⌊2kn−2k−1​⌋ штук

Осталось разобраться с особыми узлами. Пусть двоичное представление числа nnn

n=(bh bh−1 bh−2 ⋯ b2,b1,b0)2где h=⌊log⁡2n⌋n = \bigl( b_{h} \, b_{h-1} \, b_{h-2} \, \dotsm \, b_2, b_1, b_0 \bigr)_2 \quad \text{где}~ h = \lfloor \log_2 n \rfloorn=(bh​bh−1​bh−2​⋯b2​,b1​,b0​)2​где h=⌊log2​n⌋

Тогда размеры особых поддеревьев можно явно выразить через эти цифры. Напишу сверху вниз

(1 bh−1 bh−2 ⋯ b2 b1 b0)2(1 bh−2 ⋯ b2 b1 b0)2⋯(1 b1 b0)2(1 b0)212 \bigl( 1 \, b_{h-1} \, b_{h-2} \, \dotsm \, b_2 \, b_1 \, b_0 \bigr)_2 \quad \bigl( 1 \, b_{h-2} \, \dotsm \, b_2 \, b_1 \, b_0)_2 \quad \cdots \quad \bigl(1 \, b_1 \, b_0 \bigr)_2 \quad \bigl( 1 \, b_0 \bigr)_2 \quad 1_2(1bh−1​bh−2​⋯b2​b1​b0​)2​(1bh−2​⋯b2​b1​b0​)2​⋯(1b1​b0​)2​(1b0​)2​12​

Теперь мы можем посчитать количество всевозможных куч на nnn элементах. Пусть svs_vsv​ — размер поддерева с корнем в vvv. Тогда всего куч

Пирамидальная сортировка

С помощью кучи можно реализовать прекрасный алгоритм сортировки, называемый пирамидальной сортировкой. Названия действительно не совпадают. По-русски куча это пирамида, но такое название приоритетной очереди не прижилось в профессиональных кругах. А сортировка, работающая с помощью кучи, по-прежнему называется пирамидальной. В английском языке все хорошо: структура данных heap, сортировка heap sort.

Нам дали массив, который нужно отсортировать. Применим операцию heapify, превратив массив в кучу на максимум за 4n+O(log⁡n)4n + O(\log n)4n+O(logn) сравнений и 2n+O(log⁡n)2n + O(\log n)2n+O(logn) обменов. Затем будем извлекать максимум из кучи, пока та не кончится.

Heapify работает без дополнительной памяти, а вытаскиваемый максимум помещается в самый конец массива. Поэтому, можно с каждой операцией извлечения максимума уменьшать логический размер массива. В итоге максимумы будут складываться с конца в порядке убывания, и в итоге мы получим отсортированный массив.

function heap_sort(mutable array h): int n = length(h) heapify() repeat n times: swap h[0] = h[n-1] n -= 1 shift_down(0)

Если в алгоритм пирамидальной сортировки даётся случайная перестановка чисел (1,2,…,n)(1, 2, \dotsc, n)(1,2,…,n), то при heapify может равновероятно получиться любая из n!/∏vsvn! \bigm/ \prod\limits_{v} s_vn!/v∏​sv​ возможных куч.

Построение кучи, как мы уже выяснили, требует 4n+O(log⁡n)4n + O(\log n)4n+O(logn) сравнений и 2n+O(log⁡n)2n + O(\log n)2n+O(logn) обменов. После этого мы nnn раз извлекаем максимум, который находится в корне, на высоте ⌊log⁡2n⌋+1\lfloor \log_2 n \rfloor + 1⌊log2​n⌋+1. Эта операция просто вызывает просеивание вниз, а значит требует ⌊log⁡2n⌋\lfloor \log_2 n \rfloor⌊log2​n⌋ обменов и 2⌊log⁡2n⌋2 \lfloor \log_2 n \rfloor2⌊log2​n⌋ сравнений. В итоге пирамидальная сортировка требует

максимум2n⌊log⁡2n⌋+4n+O(log⁡n)  сравненийиn⌊log⁡2n⌋+2n+O(log⁡n)  обменов \text{максимум} \quad 2 n \lfloor \log_2 n \rfloor + 4n + O(\log n) ~~\text{сравнений} \quad \text{и} \quad n \lfloor \log_2 n \rfloor + 2n + O(\log n) ~~\text{обменов} максимум2n⌊log2​n⌋+4n+O(logn)  сравненийиn⌊log2​n⌋+2n+O(logn)  обменов

Огрубив, говорим, что пирамидальная сортировка работает за Θ(nlog⁡n)\Theta(n \log n)Θ(nlogn).

ddd-арная куча

Есть смысл рассматривать не только бинарные деревья для представления кучи, а вообще любые ddd-арные. Высота таких деревьев ⌊log⁡d(n(d−1))⌋+1\bigl\lfloor \log_d \bigl( n (d-1) \bigr) \bigr\rfloor + 1⌊logd​(n(d−1))⌋+1.

Точно так же, как и для двоичной кучи, у ddd-арной кучи есть свойство кучи: значение любого узла больше значений всех его поддеревьев или, что то же самое, больше значений всех его детей.

Хранить полное ddd-арное дерево можно точно так же в массиве. У элемента с индексом iii детьми являются элементы с индексами di+1,di+2,…,di+ddi+1, d_i+2, \dotsc, di+ddi+1,di​+2,…,di+d, а родителем — элемент с индексом ⌊(i−1)/d⌋\lfloor (i-1) / d \rfloor⌊(i−1)/d⌋.

Все операции: просеивание, вставка, извлечение максимума и построение выполняются точно так же, как и с обычной двоичной кучей.

Просеивание вверх требует максимум ⌊log⁡d(n(d−1))⌋\bigl\lfloor \log_d \bigl( n (d-1) \bigr) \bigr\rfloor⌊logd​(n(d−1))⌋ обменов и сравнений, а просеивание вниз требует максимум ⌊log⁡d(n(d−1))⌋\bigl\lfloor \log_d \bigl( n (d-1) \bigr) \bigr\rfloor⌊logd​(n(d−1))⌋ обменов и максимум d⌊log⁡d(n(d−1))⌋d \bigl\lfloor \log_d \bigl( n (d-1) \bigr) \bigr\rfloord⌊logd​(n(d−1))⌋ сравнений.

Разберем операцию построения ddd-арной кучи с помощью heapify. Количество сравнений для операции heapify равно

∑h=1⌊log⁡d((n(d−1))⌋+1⌈ndh⌉⋅(dh−d)=dn∑h=1⌊log⁡d((n(d−1))⌋+1h−1dh=d2(d−1)2⋅n+O(log⁡dn) \sum\limits_{h=1}^{{\small\lfloor} \log_d {\small(} \mathstrut n (d-1) {\small)} {\small\rfloor} + 1} \Bigl\lceil \frac{n}{d^h} \Bigr\rceil \cdot (dh - d) = dn \sum\limits_{h=1}^{{\small\lfloor} \log_d {\small(} \mathstrut n (d-1) {\small)} {\small\rfloor} + 1} \frac{h-1}{d^h} = \frac{d^2}{(d-1)^2} \cdot n + O(\log_d n) h=1∑⌊logd​((n(d−1))⌋+1​⌈dhn​⌉⋅(dh−d)=dnh=1∑⌊logd​((n(d−1))⌋+1​dhh−1​=(d−1)2d2​⋅n+O(logd​n)

А количество обменов в ddd раз меньше, d/(d−1)2⋅n+O(log⁡dn)d / (d-1)^2 \cdot n + O(\log_d n)d/(d−1)2⋅n+O(logd​n).

То есть операции выполняются за время

  • вставка — O(log⁡dn)O(\log_d n)O(logd​n)
  • извлечение максимума — O(dlog⁡dn)O(d \log_d n)O(dlogd​n)
  • построение ddd-арной кучи — O(n)O(n)O(n)

При этом эффективность ddd-арной кучи сильно растёт только при маленьких ddd.
