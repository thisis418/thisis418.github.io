Сортировка вставками

Мы решаем задачу сортировки: нам нужно упорядочить ключи K1,K2,…,KnK_1, K_2, \dotsc, K_nK1​,K2​,…,Kn​ по возрастанию относительного какого-то линейного порядка.

Метод простых вставок

Допустим, мы уже отсортировали j−1j-1j−1 записей так, что

K1⩽K2⩽⋯⩽Kj−2   ⁣⩽   ⁣Kj−1K_1 \le K_2 \le \dotsb \le K_{j-2} \;\! \le \;\! K_{j-1}K1​⩽K2​⩽⋯⩽Kj−2​⩽Kj−1​

Теперь нам нужно вставить следующий ключ KjK_jKj​. Будем сравнивать его по очереди с ключами Kj−1,Kj−2,…K1K_{j-1}, K_{j-2}, \dotsc K_1Kj−1​,Kj−2​,…K1​, пока не обнаружим, что его нужно вставить между ключами KiK_iKi​ и Ki+1K_{i+1}Ki+1​. Сдвинем все ключи, начиная с Ki+1K_{i+1}Ki+1​, на одну позицию и вставим запись KjK_jKj​ на место i+1i+1i+1.

Другими словами, бы каждый раз берем новый ключ и вставляем его в правильное место.

function insertion_sort(mutable array a[n]): for int j = 1; j < n; j++: int i = j - 1 key = a[j] while i >= 0 and a[i] > key: a[i+1] = a[i] i -= 1 a[i+1] = key

Количество сравнений в этом алгоритме равно числу инверсий перестановки (K1,K2,…,Kn)(K_1, K_2, \dotsc, K_n)(K1​,K2​,…,Kn​) плюс количество выходных сравнений в цикле while — n−1n-1n−1

Бинарные и двухпутевые вставки

В алгоритме простых вставок новый jjj-й элемент сравнивался в среднем с (j−1)/2(j-1)/2(j−1)/2 уже отсортированными элементами по порядку. В итоге общее количество сравнений в среднем равно ∑j   ⁣=   ⁣2n(j−1)/2=(n2−n)/4=O(n2)\sum\limits_{j \;\! = \;\! 2}^n (j-1)/2 = (n^2 - n) / 4 = O(n^2)j=2∑n​(j−1)/2=(n2−n)/4=O(n2).

Можно использовать то, что элементы, в которые мы хотим вставить новый ключ, уже отсортированы. Будем искать место вставки с помощью бинарного поиска. До конца это проблему не решит, ведь кроме поиска места вставки нужно её произвести. Но даже так это неплохой буст к производительности

Итого мы для каждого элемента с номером j+1j+1j+1 проводим бинарный поиск в подмассиве размера jjj (все, что находится спереди). В итоге максимальное количество сравнений
