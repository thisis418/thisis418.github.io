Итераторы

Посмотрим на read-only линейные списки — такие линейные списки, которые нам нужно только читать.

Пусть нам нужен read-only линейный список, в котором по порядку хранятся натуральные числа от 111 до 10910^9109. Конечно, хранить почти гигабайт данных в каком-нибудь связном списке или массиве не выгодно.

Единственная операция, которую мы хотим совершать — чтение. Получается, нам достаточно определить только первый элемент, способ для любого элемента получить следующий и какой-то индикатор конца списка.

В этом примере первый элемент 111. Для получения элемента, следующего за текущим, нужно к текущему прибавить 111. Конца списка мы достигнем тогда, когда дойдем до элемента 10910^9109.

Read-only линейный список не обязательно должен содержать в себе все свои элементы.

Итератор

Итератор (iterator) — изменяемый объект, хранящий свое состояние и функция next(), которая возвращает или следующий элемент, или stop, если итератор закончится.

Двусторонний итератор (doubly-iterator, next-prev iterator) — изменяемый объект, хранящий свое состояние, функция next(), которая возвращает следующий элемент и функция previous(), которая возвращает предыдущий элемент.

Итератор и итерируемый объект

Существует несколько разных и полностью адекватных способов определить всё это. Здесь я придерживаюсь классического высокоуровневого определения.

Поэтому итератор это итератор, а итерируемый объект (iterable) — любой объект, который можно нормально превратить в итератор.

Например любой массив и любая строка являются итерируемыми объектами, но не итераторами.

Функциональное представление итерируемого

Любой итератор можно сконструировать с помощью обычной функции и инструкции yield. Эта операция «возвращает» значение из функции, но выполнение функции не прекращается. Функция выполняется до тех пор, пока она не закончится или пока не встретится инструкция stop.

Фактически, итератор не может возвращать значения, ведь в этом слове подразумевается завершение работы функции. В разговорной речи для этого используется термин «йелдить». Здесь я все-же буду писать слово «возвращать».

Посмотрим на итератор, который возвращает квадраты чисел от 111 до 100100100.

int current = 1 function next() -> int: if current > 100: stop else: to_return = current ** 2 current += 1 return to_return

Этот итератор можно легко записать в функциональной форме

iterator squares: int current = 1 while current <= 100: yield current ** 2 current += 1 stop

Обращаю внимание на то, что squares — это объект. Это не функция, возвращающая итератор.

Теперь, имея на руках итератор, можно по нему итерироваться.

Итерирование по итератору — последовательные получения следующего элемента, то есть последовательные вызовы функции next(), пока однажды мы не получим stop.

iterator i do: value = i.next() ... while value != stop

Этот шаблонный код обычно сокращается до привычного всем программистам for

iterator i for each value of i: ...

Генераторы

В чистом виде итераторы нужны не так часто, как функции, создающие итераторы.

Генератор

Генератор — функция, создающая и возвращающая итератор.

Эта конструкция настолько часто встречается, что для нее не жалко определить свой синтаксис.

Итак, чистая функция принимает аргументы ...arguments, создает итератор и возвращает его.

function gen(arguments): iterator i: ... return i

Выражение ..., определяющее итератор, пишется сразу в определении генератора. Синтаксис следующий

generator gen(arguments): ...

Пусть нам нужно создать итератор, который возвращает все целые числа от aaa до bbb.

generator range(int a, int b) -> int: int i = a while i <= b: yield i i += 1 stop

Получается, что огрызки кода for i = a; i <= b; i++ и for i of range(a, b) имеют абсолютно одинаковый смысл и могут друг друга заменять без потери функционала.

Важно понимать, что вариант с использованием генератора сильно дольше. Там создаются новые локальные переменные, в стек кладется еще одна функция, внутри происходит b−a+2b-a+2b−a+2 высокоуровневых вызовов и еще куча другого оверхеда.

Итераторы и Python

В питоне есть только итераторы. Питоновская конструкция for i in iterable эквивалентна нашей конструкции for each i of iterable.
