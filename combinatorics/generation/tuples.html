Генерация кортежей

Начнём с простого. Пусть нам нужно сгенерировать все 2n2^n2n бинарных строк длины nnn. Точнее, нам нужно сделать итератор, который выдает нам все возможные битовые строки по одному разу.

Битовую строку можно рассматривать как число в двоичной системе счисления. Тогда нам достаточно просто перебирать все числа от 000 до 2n−12^n-12n−1 и переводить их в двоичную систему счисления:

generator binary_tuples(int n) -> tuple[{0, 1}]: for int i = 0; i < 2 ** n; i++: yield convert(n, to base 2)

Точно также можно поступить и когда нам нужно сгенерировать ddd-арные строки длины nnn. Это строки из множества {0,1,2,…,d−1,d}n\{0, 1, 2, \dotsc, d-1, d\}^n{0,1,2,…,d−1,d}n.

Представляем все числа от 000 до dn−1d^n-1dn−1 в системе счисления по основанию ddd и выдаём их по порядку.

generator tuples(int n, int d) -> tuple[int]: for int i = 0; i < d ** n; i++: yield convert(n, to base d)

Такой лобовой метод, конечно, имеет право на существования, но он немного неэффективен.

Перевод числа nnn в систему счисления по основанию ddd требует ⌊log⁡dn⌋+1\lfloor \log_d n \rfloor + 1⌊logd​n⌋+1 итераций, а каждая итерация выполняет одно деление на ddd и одно взятие по модулю ddd. Итого для полной генерации количество итераций равно

1+∑k=1dn−1(⌊log⁡dn⌋+1)=n⋅dn−dn−1d−1+11 + \sum\limits_{k=1}^{d^n-1} \bigl( \lfloor \log_d n \rfloor + 1\bigr) = n \cdot d^n - \frac{d^n - 1}{d - 1} + 11+k=1∑dn−1​(⌊logd​n⌋+1)=n⋅dn−d−1dn−1​+1

На самом деле нам не нужно каждый раз переводить число в систему счисления по основанию ddd. Следующее число отличается от предыдущего только на 111, поэтому можно просто прибавлять 111, но уже в позиционном представлении числа. Добавление происходит прям также, как при сложении в столбик: переносим разряды когда надо.

Смешанная система счисления

Вообще, для такой задачи генерации есть обобщение: сгенерировать все возможные кортежи длины nnn, где на позиции jjj стоит число aja_jaj​, которое 0⩽aj<mj0 \le a_j < m_j0⩽aj​<mj​.

Получается, задача генерации сводится к задаче «добавления 111» к «числу» a1 a2 a3 ⋯ an−1 ana_1 \, a_2 \, a_3 \, \dotsm \, a_{n-1} \, a_na1​a2​a3​⋯an−1​an​, записанному в смешанной позиционной счисления по основанию (m1,m2,m3,…,mn−1,mn)(m_1, m_2, m_3, \dotsc, m_{n-1}, m_n)(m1​,m2​,m3​,…,mn−1​,mn​):

[a1a2a3⋯an−1anm1m2m3⋯mn−1mn]\bmatrix{a_1 & a_2 & a_3 & \cdots & a_{n-1} & a_n \\ m_1 & m_2 & m_3 & \cdots & m_{n-1} & m_n}[a1​m1​​a2​m2​​a3​m3​​⋯⋯​an−1​mn−1​​an​mn​​]

Для удобства введем дополнительные переменные a0a_0a0​ и m0m_0m0​.

generator radix_tuples(tuple[int] m[n]) -> tuple[int]: array[int] a = [0, 0, ..., 0] yield tuple(a) forever: a[-1] += 1 for int i = n - 1; i >= 0; i--: if a[i] >= m[i]: if i == 0: stop a[i] = 0 a[i - 1] += 1 else: break yield tuple(a)

Всего будет сгенерировано ∏j=1nmj\prod\limits_{j=1}^n m_jj=1∏n​mj​ кортежей в лексикографическом порядке.
