Тесты на простоту

Определять, является ли число простым можно с помощью решёт, но этот способ нельзя назвать полноценным тестом на простоту, так как попутно мы проверяем на простоту все числа до исходного. Тесты на простоту делятся на два типа: истинные и вероятностные. Истинные тесты точно отвечают, является ли число простым, вероятностные же могут дать ответ, который будет верным с некоторой возможностью ошибки. Зачастую такие тесты используют в связке: сначала проверяют число вероятностным тестом, а потом подтверждают это детерменированным.

Вероятностные тесты

Тест Ферма

Тест Ферма основывается на малой теореме Ферма, которая гласит

a⊥p  ⟹  ap−1≡1(modp)где p простоеa \rprime p \implies a^{p-1} \equiv 1 \pmod p \quad\text{где}~ p ~\text{простое}a⊥p⟹ap−1≡1(modp)где p простое

Чтобы узнать, является ли nnn простым числом, нужно взять случайное число a<na < na<n и проверить, выполняется ли an−1≡1(modn)a^{n-1} \equiv 1 \pmod nan−1≡1(modn). Число aaa называется свидетелем простоты. Если равенство не выполняется, то с полной уверенностью можно сказать, что число составное. Если же nnn прошло тест, то всё равно есть вероятность, что оно не простое.

Заметьте, что в малой теореме Ферма импликация идёт только в одну сторону: из взаимной простоты следует сравнение, но обратная импликация не всегда верна. Составные числа nnn, удовлетворяющие сравнению an−1≡1(modn)a^{n-1} \equiv 1 \pmod nan−1≡1(modn) для всех целых aaa, взаимнопростых с nnn, называются числами Кармайкла. Числа Кармайкла успешно проходят тест Ферма, хотя являются составными.

Давайте посмотрим на наименьшее число Кармайкла — 561=3⋅11⋅17561 = 3 \cdot 11 \cdot 17561=3⋅11⋅17. Из китайской теоремы об остатках следует, что Z561=Z3×Z11×Z17\ZZ_{561} = \ZZ_{3} \times \ZZ_{11} \times \ZZ_{17}Z561​=Z3​×Z11​×Z17​. Возьмём aaa взаимнопростое со всеми делителями 561561561, то есть

a⊥3  ⟹  a2≡31a⊥11  ⟹  a10≡111a⊥17  ⟹  a16≡171\align{ a \rprime 3 &\implies a^2 \eqmod{3} 1 \\ a \rprime 11 &\implies a^{10} \eqmod{11} 1 \\ a \rprime 17 &\implies a^{16} \eqmod{17} 1 }a⊥3a⊥11a⊥17​⟹a2≡31⟹a10≡111⟹a16≡171​

222, 101010 и 161616 являются делителями 560560560, значит a560≡5611a^{560} \eqmod{561} 1a560≡5611.

Такие числа встречаются довольно редко, но их бесконечно много.

Вероятность ошибки одного теста будет ε⩽φ(n)/n\varepsilon \le \varphi(n)/nε⩽φ(n)/n. Выполнив тест Ферма kkk раз, получим вероятность верного ответа 1−εk1 - \varepsilon^k1−εk.

function fermat_test(int n, int k) -> bool: for int i = 0; i < k; i++: select a = random uniform int [2, n-1] if pow(a, n-1) % n != 1: return false return true

Используя алгоритмы быстрого возведения в степень по модулю, можно получить время работы O(klog⁡2n⋅log⁡log⁡n⋅log⁡log⁡log⁡n)O(k \log^2n \cdot \log \log n \cdot \log \log \log n)O(klog2n⋅loglogn⋅logloglogn).

Тест Соловея-Штрассена

Тест Соловея-Штрассена предполагает проверку квадратичных вычетов, для этого нам понадобится критерий Эйлера.

Критерий Эйлера

ppp — простое, p>2p > 2p>2 и a⊥pa \rprime pa⊥p

a(p−1)/2≡1(modp) когда a квадратичный вычет по модулю pa(p−1)/2≡−1(modp) когда a квадратичный невычет\align{&a^{(p-1)/2} \equiv 1 \pmod p ~ \text{когда} ~ a ~ \text{квадратичный вычет по модулю} ~ p \\ &a^{(p-1)/2} \equiv -1 \pmod p ~ \text{когда} ~ a ~ \text{квадратичный невычет}}​a(p−1)/2≡1(modp) когда a квадратичный вычет по модулю pa(p−1)/2≡−1(modp) когда a квадратичный невычет​

Рассмотрим три утверждения:

  1. Существует такой xxx, что x2≡a(modp)x^2 \equiv a \pmod px2≡a(modp)
  2. a(p−1)/2≡1(modp)a^{(p-1)/2} \equiv 1 \pmod pa(p−1)/2≡1(modp)
  3. a(p−1)/2≡−1(modp)a^{(p-1)/2} \equiv -1 \pmod pa(p−1)/2≡−1(modp)

Из p>2p > 2p>2 следует, что 1≢p−1(modp)1 \notequiv p-1 \pmod p1≡p−1(modp), а это то же самое, что 1≢−1(modp)1 \notequiv -1 \pmod p1≡−1(modp), поэтому утверждения 222 и 333 не могут выполняться одновременно.

Пусть существует xxx такое, что x2≡a(modp)x^2 \equiv a \pmod px2≡a(modp). Возведём левую и правую часть в (p−1)/2(p-1)/2(p−1)/2 степень и получим xp−1≡a(p−1)/2(modp)x^{p-1} \equiv a^{(p-1)/2} \pmod pxp−1≡a(p−1)/2(modp). По малой теореме Ферма xp−1≡1(modp)x^{p-1} \equiv 1 \pmod pxp−1≡1(modp), а значит a(p−1)/2≡1(modp)a^{(p-1)/2} \equiv 1 \pmod pa(p−1)/2≡1(modp). Получается, что утверждения 111 и 222 всегда выполняются вместе.

Рассмотрим последовательность

1, 2, …, p−12, p+12, …, p−11, \, 2, \, \dotsc, \, \frac{p-1}{2} , \, \frac{p+1}{2} , \, \dotsc , \ p-11,2,…,2p−1​,2p+1​,…, p−1

Из −1≡p−1(modp)-1 \equiv p-1 \pmod p−1≡p−1(modp) следует

12≡(p−1)2, 22≡(p−2)2, …, (p−12)2≡(p+12)2(modp)1^2 \equiv (p-1)^2, \, 2^2 \equiv (p-2)^2, \, \dotsc, \, \left( \frac{p-1}{2} \right)^2 \equiv \left( \frac{p+1}{2} \right)^2 \quad \pmod p12≡(p−1)2,22≡(p−2)2,…,(2p−1​)2≡(2p+1​)2(modp)

Получается, что сущетвует ровно (p−1)/2(p-1)/2(p−1)/2 квадратов по модулю ppp. Обозначим их a1, a2, ⋯ , a(p−1)/2a_1, \, a_2, \, \dotsm, \, a_{(p-1)/2}a1​,a2​,⋯,a(p−1)/2​. Если aaa равно aja_jaj​, то сравнение 111 имеет решение, а это значит, что и второе выполняется для любого aja_jaj​. Следовательно, сравнение 222 имеет ровно (p−1)/2(p-1)/2(p−1)/2 решений, отсюда же следует, что 333 также имеет (p−1)/2(p-1)/2(p−1)/2 решений, при каждом из которых первое сравнение не выполняется.

Если число nnn простое, то n−1=2s⋅dn-1 = 2^s \cdot dn−1=2s⋅d, где sss целое, а ddd нечётное. Очевидно, что an−1≡n±1{\sqrt {a^{n-1}}} \eqmod{n} \pm 1an−1​≡n±1. Значит, если мы возьмём случайное aaa, то оно должно быть либо квадратичным вычетом, либо невычетом по модулю nnn. Это можно выразить через символы Якоби следующим образом:

an−1≡1иa(n−1)/2≡(ap)(modn)a^{n-1} \equiv 1 \quad \text{и} \quad a^{(n-1)/2} \equiv \legendre{a}{p} \pmod nan−1≡1иa(n−1)/2≡(pa​)(modn)
function solovay_strassen_test(int n, int k) -> bool: for int i = 0; i < k; i++: select a = random uniform int [2, n-1] if pow(a, (n-1)/2) % n != jacobi(a, p): return false return true

Асимптотическая сложность этого алгоритма равняется O(klog⁡3n)O(k\log^3 n)O(klog3n), а вероятность ошибки ε⩽1/2k\varepsilon \le 1/2^kε⩽1/2k.

Тест Миллера-Рабина

Давайте усовершенствуем разобранные нами тесты. В предыдущем пункте мы выяснили, что n−1=2s⋅dn-1=2^s \cdot dn−1=2s⋅d, значит мы можем брать корень из aaa, пока не выполнится одно из двух:

ad≡±1илиa2r⋅d≡−1(modn), где r<sa^d \equiv \pm 1 \quad \text{или} \quad a^{2^r \cdot d} \equiv -1 \quad \pmod n, ~ \text{где} ~ r < sad≡±1илиa2r⋅d≡−1(modn), где r<s

Если при взятии корня мы получили сравнимость по модулю nnn с чем-то кроме ±1\pm 1±1, то число точно составное. Вероятность ошибки этого теста ε⩽1/4\varepsilon \le 1/4ε⩽1/4. Значит, повторив тест Миллера-Рабина kkk раз, мы получим вероятность верного ответа не меньше 1−1/4k1 - 1/4^k1−1/4k, что является очень хорошим результатом.

function miller_rabin_test(int n, int k) -> bool: for int i = 0; i < k; i++: select a = random uniform int [2, n-1] for int j = n-1; j % 2 == 0; j /= 2: if pow(a, j) % n == n-1: break else if pow(a, j) % n != 1: return false return true

Тест Миллера-Рабина способен определять некоторые числа Кармайкла, например, 561561561, но вот 172917291729, третье число Кармайкла, успешно проходит тест.

Сложность этого алгоритма составляет O(klog⁡3n)O(k \log^3 n)O(klog3n).

Тест Фробениуса

Для использования теста Фробениуса нужно рассмотреть такое понятие как квадратичная иррациональность. Квадратичной иррациональностью будем называть число z=a+bcz = a +b \sqrt cz=a+bc​, где aaa, bbb и ccc — целые и ccc свободно от квадратов. Сопряжённым к zzz будет zˉ=a−bc\bar{z} = a - b \sqrt czˉ=a−bc​. А остаток от деления определим как zmod  n=(amod  n)+(bmod  n)⋅cz \mod n = (a \mod n) + (b \mod n) \cdot \sqrt czmodn=(amodn)+(bmodn)⋅c​.

Теорема Фробениуса

Пусть nnn — простое, символ Якоби (an)=−1\legendre{a}{n} = -1(na​)=−1 и z∈Zn[c]z \in \ZZ_n [\sqrt c]z∈Zn​[c​], тогда

zn≡zˉ(modn)z^n \equiv \bar{z} \pmod nzn≡zˉ(modn)

Сутью теста Фробениуса является подбор подходящих коэффициентов aaa, bbb и ccc. Начать проще всего с последнего. Чтобы выполнялась теорема Фробениуса, нам нужно такое ccc, что (cn)=−1\legendre{c}{n} = -1(nc​)=−1, в качестве значения ccc возьмём −1-1−1 или наименьшее простое число, удовлетворяющее условию. Если c⩽2c \le 2c⩽2, то z=2+cz = 2 + \sqrt cz=2+c​, иначе z=1+cz = 1 + \sqrt cz=1+c​. Если zn≡zˉ(modn)z^n \equiv \bar{z} \pmod nzn≡zˉ(modn), то число простое с вероятностью 1−7710−11 - 7710^{-1}1−7710−1. Несмотря на то, что тест вероятностный, на данный момент неизвестно ни одно составное число, проходящее его, и строго доказано, что таких чисел меньше 2602^{60}260 не существует.

function frobenius_test(int n) -> bool: tuple [int, complex] z for int c in [-1, 2, 3, 5, 7, ...]: if jacobi(c, n) == -1: if c <= 2: z.first = 2 z.second = sqrt(c) else: z.first = 1 z.second = sqrt(c) if pow(z, n) % n == !z: return true else: return false

Тест Люка
