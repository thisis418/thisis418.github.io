Здесь мы будем находить все простые числа от 111 до nnn.

Решето Эратосфена

Решето Эратосфена — алгоритм нахождения списка простых чисел.

Идея простая. Запишем ряд чисел 2,3,4,…,n−1,n2, 3, 4, \dotsc, n-1, n2,3,4,…,n−1,n. Вычеркнем из него все числа, делящиеся на 222, кроме, конечно, 222. Затем вычеркнем все числа, делящиеся на 333, кроме 333. Число 444 уже вычеркнуто, и все числа, кратные 444, тоже. Поэтому идём дальше. Вычеркиваем все числа, делящиеся на 555, кроме 555. Ну и продолжаем так, пока не дойдем до конца.

Итого, мы для каждого простого числа вычёркиваем все дальнейшие числа, кратные ему.

Реализовывать программно это можно по-разному, но самый простой способ — завести битовый массив is_prime. Если число iii вычеркнуто, то is_prime[i−2]=0\code{is\_prime}[i-2] = 0is_prime[i−2]=0. Вообще можно оставить лишние 222 бита в начале и сделать нормальную индексацию.

input const int n array[bool] is_prime[n+1] = [true, true, ..., true] for int i = 2; i <= n; i++: if is_prime[i]: for int j = 2 * i; j <= n; j += i: is_prime[j] = false

Есть несколько очевидных оптимизаций, которые стоит сразу же применить.

Во-первых, начинать вычёркивать кратные числа ppp можно с p2p^2p2, а не с 2p2p2p. Ведь все составные числа, кратные ppp и меньшие p2p^2p2 были вычеркнуты до этого, потому что у них был делитель, меньший p2/p=pp^2/p = pp2/p=p.

Во-вторых, внешний цикл можно сделать до n\sqrt{n}n​. Пусть ppp — простое число, большее n\sqrt{n}n​. Его наименьшее кратное, которое можно вычеркнуть — p2p^2p2. Оно будет находиться за диапазоном поиска простых, ведь p2>np^2 > np2>n.

С учётом всех оптимизаций, получаем код

input const int n array[bool] is_prime[n+1] = [true, true, ..., true] for int i = 2; i * i <= n; i++: if is_prime[i]: for int j = i * i; j <= n; j += i: is_prime[j] = false

Проанализируем время работы алгоритма. Будем измерять число установок в битовом массиве is_prime.

Для каждого простого числа ppp в диапазоне от 222 до n\sqrt{n}n​ мы сделаем ⌊n/p⌋−(p−1)\lfloor n / p \rfloor - (p-1)⌊n/p⌋−(p−1) установок. Количество чисел в диапазоне от 111 до nnn, которые кратны ppp, равно ⌊n/p⌋\lfloor n/p \rfloor⌊n/p⌋. Тогда общее количество установок, сделанное алгоритмом, равно

где M≈0.261 497M \approx 0.261\,497M≈0.261497 — константа Мейсселя-Мертенса.

Линейное решето Эратосфена

Основная проблема решета Эратосфена в том, что мы неизбежно помечаем некоторые числа как составные несколько раз. Мы помечаем число как составное столько раз, сколько у него различных простых делителей.

Пусть d(k)   ⁣=def   ⁣min⁡p простое{p\k}d(k) \defeq \min\limits_{p ~\text{простое}} \{ p \divides k\}d(k)=defp простоеmin​{p\k} — минимальный простой делитель числа kkk. Любое число kkk тогда представимо в виде k=d(k)⋅rk = d(k) \cdot rk=d(k)⋅r, где у rrr нет простых делителей, меньших d(k)d(k)d(k).

Давайте для каждого числа kkk считать d(k)d(k)d(k). Заведем массив d, в котором будем хранить найденные значения.
