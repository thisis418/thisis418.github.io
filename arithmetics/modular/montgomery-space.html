Пространство Монтгомери

В модулярной арифметике основная операция — само вычисление остатка от деления числа на модуль. Эта операция очень дорогая — целых 15–30 тактов.

Вместо работы с числами в обычном виде a mod Ma \bmod MamodM, Питер Монтгомери предложил переходить в специальное представление — представление Монтгомери, где умножение выполняется быстрее, чем классическое модульное умножение.

Представление Монтгомери можно построить по любому модулю MMM. Для этого представления надо сначала выбрать число RRR, которое будет взаимнопросто с MMM: R⊥MR \rprime MR⊥M.

Обычное число xxx преобразуется в представление Монтгомери следующим образом

x‾=x⋅R mod M\overline{x} = x \cdot R \bmod Mx=x⋅RmodM

Для того, чтобы вычислить это преобразование, понадобится выполнить ту самую дорогую операцию взятия по модулю. К счастью, это придётся проделать только два раза для каждого числа: чтобы перевести в пространство Монтгомери и чтобы перевести обратно. Все промежуточные вычисления выполняются без этой дорогой операции.

Умножение Монтгомери

Умножение Монтгомери ∗*∗ не совпадает с обычным модульном умножением ⋅\cdot⋅. Для того, чтобы после умножения получилось число, которое является представлением Монтгомери произведения операндов, нужно дополнительно домножить на R−1R^{-1}R−1:

a‾∗b‾   ⁣=def   ⁣a⋅b‾=(a⋅b)⋅R mod M=(a‾⋅b‾)⋅R−1 mod M\overline{a} * \overline{b} \defeq \overline{a \cdot b} = (a \cdot b) \cdot R \bmod M = (\overline{a} \cdot \overline{b}) \cdot R^{-1} \bmod Ma∗b=defa⋅b=(a⋅b)⋅RmodM=(a⋅b)⋅R−1modM

Вся прелесть в представлении Монтгомери в том, что операцию x⋅R−1 mod Mx \cdot R^{-1} \bmod Mx⋅R−1modM можно выполнить без взятия по модулю. Эта операция называется сокращение Монтгомери (Montgomery reduction).

Расширенный алгоритм Евклида позволяет найти два числа R−1R^{-1}R−1 и M′M'M′ такое, что

R⋅R−1+M⋅M′=1R \cdot R^{-1} + M \cdot M' = 1R⋅R−1+M⋅M′=1

Тогда можно получить способ вычисления x⋅R−1x \cdot R^{-1}x⋅R−1

x⋅R−1=x⋅R⋅R−1/R==x⋅(1−M⋅M′)/R≡≡(x−x⋅M⋅M′+k⋅R⋅M)/R(modM)≡≡(x−(x⋅M′−k⋅R)⋅M)/R(modM)\align{ x \cdot R^{-1} &= x \cdot R \cdot R^{-1} / R = \\ &= x \cdot (1 - M \cdot M') / R \equiv \\ &\equiv (x - x \cdot M \cdot M' + k \cdot R \cdot M) / R \pmod{M} \equiv \\ &\equiv \bigl( x - (x \cdot M' - k \cdot R) \cdot M \bigr) / R \pmod{M} }x⋅R−1​=x⋅R⋅R−1/R==x⋅(1−M⋅M′)/R≡≡(x−x⋅M⋅M′+k⋅R⋅M)/R(modM)≡≡(x−(x⋅M′−k⋅R)⋅M)/R(modM)​

Здесь kkk — любое целое число. Давайте выберем k=⌊x⋅M′/R⌋k = \lfloor x \cdot M' / R \rfloork=⌊x⋅M′/R⌋. Тогда x⋅M′−k⋅R=x⋅M′ mod Rx \cdot M' - k \cdot R = x \cdot M' \bmod Rx⋅M′−k⋅R=x⋅M′modR, и

x⋅R−1≡(x−x⋅M′ mod R⋅M)/R(modM)x \cdot R^{-1} \equiv (x - x \cdot M' \bmod R \cdot M) / R \pmod{M}x⋅R−1≡(x−x⋅M′modR⋅M)/R(modM)

Осталась небольшая проблемка: x/R<Mx/R < Mx/R<M и x⋅M′ mod R⋅M/R<Mx \cdot M' \bmod R \cdot M / R < Mx⋅M′modR⋅M/R<M, поэтому результат вычитания находится в интервале (−M,M)(-M, M)(−M,M). Для решения этой проблемки надо просто добавить MMM к результату, если надо.

Получаем уже полный алгоритм усечения

const int M, R const int Mr = inverse(M, mod R) function reduce(int x): int q = x * Mr % R int m = q * M int y = (x - m) / R if y < 0: return y + M else: return y

Это же усечение применяется для того, чтобы перевести число из представления Монтгомери в обычное представление.

Компьютерная реализация

Что у нас в итоге получилось? Вместо одного умножения и одного взятия по модулю мы выполняем два умножения, одно сложение, одно взятие по модулю и одно деление. Прикол в том, что мы выполняем деление не на данное нам число MMM, а на почти произвольное число RRR.

Мы можем выбрать RRR удобным для себя. Например, степень двойки! Деление на степень двойки можно реализовать через сдвиг, а взятие по модулю реализовывать вообще не нужно, если подобрать правильный тип данных.

Итого мы получаем очень эффективный и быстрый код. Здесь я взял R=232R = 2^{32}R=232.

typedef uint32_t u32; typedef uint64_t u64; constexpr u32 M = 1e9 + 7; const u32 Mr = inverse(M, 1ull << 32); u32 reduce(const u64 x) { u32 q = static_cast<u32>(x) * Mr; u64 m = static_cast<u64>(q) * M; u32 y = (x - m) >> 32; return x - m ? y + M : y; }

Если мы можем позволить себе вернуть число в диапазоне от 000 до 2n−22n-22n−2, то можно отказаться от последней проверки и всегда прибавлять MMM. В итоге получится код return y + M без ветвлений.

Можно также битовый сдвиг >> 32 выполнять раньше и вычислять ⌊x/R⌋−⌊m/R⌋\lfloor x / R \rfloor - \lfloor m / R \rfloor⌊x/R⌋−⌊m/R⌋ вместо (x−m)/R(x - m) / R(x−m)/R. Так делать можно, потому что первые 323232 бита в xxx и mmm совпадают: m=x⋅M⋅M′≡x(modR)m = x \cdot M \cdot M' \equiv x \pmod Rm=x⋅M⋅M′≡x(modR).

В нашем случае два битовых сдвига эффективнее одного. Причина в том, что при вычислении ⌊m/R⌋\lfloor m / R \rfloor⌊m/R⌋ мы считаем ((u64) q * n) >> 32, а при умножении 323232-битных чисел друг на друга процессоры x86/x64 старшие 323232 бита пишут в отдельный регистр edx.

u32 reduce(const u64 x) { u32 q = static_cast<u32>(x) * Mr; u32 m = (static_cast<u64>(q) * M) >> 32; return (x >> 32) + M - m; }

Быстрое обратное по модулю RRR

Для того, чтобы инициализировать пространство Монтгомери, нам нужно вычислить M′=M−1 mod RM' = M^{-1} \bmod RM′=M−1modR. Обычно это находится с помощью расширенного алгоритма Евклида. Но этот метод может оказаться долговатым.

Можно использовать тот факт, что RRR — степень двойки. Тогда

M⋅x≡1(mod2k)  ⟹  M⋅x⋅(2−M⋅x)≡1(mod22k)M \cdot x \equiv 1 \pmod{2^k} \implies M \cdot x \cdot (2 - M \cdot x) \equiv 1 \pmod{2^{2k}}M⋅x≡1(mod2k)⟹M⋅x⋅(2−M⋅x)≡1(mod22k)

Этот факт можно использовать для нахождения M′M'M′. Начнём с x=1x = 1x=1 как с обратного для MMM по модулю 222 и будем увеличивать степень модуля, пока не дойдем до модуля RRR. Нам понадобится для этого log⁡2R\log_2 Rlog2​R итераций.

function inverse(int M, mod int R): Mr = 1 repeat log2(R) times: Mr *= 2 - M * Mr return Mr

Код инициализации в C++ будет абсолютно такой же

struct Montgomery { u32 M, Mr; constexpr explicit Montgomery(const u32 M) : M(M), Mr(1) { for (int i = 0; i < 5; i++) Mr *= 2 - M * Mr; } ... }

Деление без деления

Вообще надо всегда по возможности отказываться от операций взятия по модулю.

Банальный пример: нужно посчитать сумму чисел в массиве a по модулю MMM, при этом все числа массива меньше модуля MMM.

Интуитивно понятный код может оказаться медленным. Вот наивная реализация

const int M = 1000000009; int sum(int *a, int n) { int s = 0; for (int i = 0; i < n; i++) { s = (s + a[i]) % M; } return s; }

Можно заметить, что раз a[i]<M\code{a}[i] < Ma[i]<M, и в начале каждой итерации s<Ms < Ms<M, то s+a[i]<2Ms + \code{a}[i] < 2Ms+a[i]<2M, а значит операцию взятия остатка можно заменить на простое условие и вычитание

const int M; int *a, int n; int s = 0; for (int i = 0; i < n; i++) { s += a[i]; if (s >= M) { s = s - M; } }
